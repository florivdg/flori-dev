# AGENTS.md

This file provides guidance to Coding Agents when working with code in this repository.

## Project Overview

Personal website for [flori.dev](https://flori.dev) — an Astro static site with Vue interactive components, deployed on Vercel.

## Commands

- **Dev server:** `bun run dev`
- **Build:** `bun run build`
- **Preview production build:** `bun run preview`
- **Type check:** `bunx astro check`
- **Type aware linting:** `bun run lint --type-aware`
- **Format:** `bunx prettier --write .`
- **Generate photo metadata:** `bun run grid:prepare` (requires `OPENAI_API_KEY`)

Use **Bun** as the package manager (not npm/yarn/pnpm). Install dependencies with `bun install`.

### Background dev server (agent-friendly)

Astro 7 detects when it runs inside an AI coding agent and automatically starts
`bun run dev` as a detached background process with JSON logging, so the dev
server never blocks the agent's terminal. A lock file (`.astro/dev.json`,
gitignored) tracks the running server's URL and PID to prevent duplicates.

Manage the background server explicitly with:

- **Start in background:** `bun run dev:bg`
- **Check status:** `bun run dev:status`
- **Stop:** `bun run dev:stop`
- **View logs:** `bun run dev:logs`

Set `ASTRO_DEV_BACKGROUND=0` to opt out of auto-backgrounding for a single run.

## Architecture

### Rendering Strategy

Static output (`output: 'static'`) with Vercel adapter. Vue components hydrate client-side via `client:only="vue"` or `client:load` directives.

### Layouts

Two-tier layout system:

- `Basic.astro` — HTML shell (fonts, meta, Plausible analytics, theme colors)
- `Layout.astro` — wraps Basic with Header + Footer; used by most pages

### Content Collections

Two collections defined in `src/content.config.ts`:

- **reads** — Blog articles as MDX files in `src/content/reads/`. Schema: title, description, pubDate, draft, tags, optional image.
- **grid** — Photo gallery. Each photo is a `.jpg` + `.json` pair in `src/content/grid/`. The JSON contains AI-generated metadata (title, alt text, location, tags) and EXIF data. Generated by `scripts/grid-vision.ts` using OpenAI Vision API.

### Markdown Pipeline

Astro 7 defaults to the Rust-based **Sätteri** Markdown processor, but this site
explicitly opts back into the `unified()` pipeline in `astro.config.mjs`
(`markdown.processor: unified({ ... })` from `@astrojs/markdown-remark`). This is
required so the `remarkExternalLinks` remark plugin and the rehype-based
Expressive Code integration keep working — don't remove it without porting those
plugins to Sätteri MDAST/HAST equivalents.

### Component Conventions

- **Astro components** (`.astro`) — static/server-rendered content
- **Vue components** (`.vue`) — interactive features (browser stats dashboard, photo gallery overlays)
- Component subdirectories group by feature: `grid/`, `reads/`, `stats/`

### Browser Stats Feature

A live dashboard at `/uses/browserstats` showing default browser usage across the author's Macs. Uses SSE (Server-Sent Events) for real-time data, Vue charts (`vue-chrts`, `@unovis/vue`), and maps macOS bundle IDs to browser names via `src/utils/browserMap.ts`.

### Styling

- Tailwind CSS 4 via Vite plugin (not PostCSS)
- Config in `tailwind.config.cjs`, theme extensions in `src/styles/tailwind.css`
- Custom fonts: Mona Sans (body), Monaspace Neon (code)
- Typography plugin for prose content
- Dark mode via `prefers-color-scheme` media queries

## Code Style

- No semicolons, single quotes, trailing commas, 2-space indent (Prettier config in `.prettierrc`)
- TypeScript in strict mode (extends `astro/tsconfigs/strict`)
- Prettier plugins: `prettier-plugin-astro`, `prettier-plugin-tailwindcss`

## Editing Guidelines

For TypeScript changes, run `bun run astro check` to verify type correctness. For Vue.js components run `bunx vue-tsc --noEmit` to check for type errors.

Run `bun run lint --type-aware` to catch linting issues.

After applying all changes, run `bunx prettier --write` on changed files to ensure consistent formatting.
